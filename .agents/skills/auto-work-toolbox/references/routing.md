# Routing Guide

按任务目标选择工具，不按技术名词堆砌选择工具。

## 架构设计

当任务涉及以下问题时，使用 `modular-arch`：
- 新增模块
- 调整模块边界
- 声明依赖方向
- 定义接口契约
- 提交前检查是否有循环依赖或跨层违规

业务上，它回答的是：
- 新能力应该放在哪个模块
- 模块之间应该如何协作
- 当前架构是否仍然可控

## 重构迁移

当任务涉及以下问题时，使用 `refactor-migration`：
- 需要一边参考旧系统，一边建设新系统
- 需要冻结旧基线并持续维护新结构
- 需要说明旧能力由谁承接、何时完成切换

业务上，它回答的是：
- 旧能力怎么平滑交接到新结构
- 迁移现在做到哪一步
- 哪些旧模块还不能下线

## 质量门禁

当任务涉及以下问题时，使用 `python-quality-gate`：
- 修改 Python 代码后需要统一检查
- 需要在提交前执行 Ruff 和 Pyright
- 需要对指定目录做局部质量校验

业务上，它回答的是：
- 当前改动是否达到可交付门槛
- 是否存在明显的静态质量风险

## 会话留痕

当任务涉及以下问题时，使用 `session-log`：
- 需要记录本次任务的原始需求
- 需要保留执行摘要和相关文件
- 需要为后续复盘提供检索入口

业务上，它回答的是：
- 这次任务做了什么
- 为什么这么做
- 后续追责或复盘时去哪里查

## 组合规则

- 纯架构设计：`modular-arch`
- 大型重构：`refactor-migration` + `modular-arch`
- 常规代码开发：按需使用相关 skill，收口时加 `python-quality-gate`
- 需要全过程可追溯：在任务开始和结束时加 `session-log`
