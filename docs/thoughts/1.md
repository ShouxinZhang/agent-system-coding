一个简单的plan-worker的agent系统

用户输入提示->
plan agent根据提示开始思考, 分析串行和并行, 最后生成一份模块化职责分明的plan, 然后dispatch并行化的worker agent任务
worker agent领取工作开始执行, 当每个worker结束工作后, 总结一份json格式的工作报告, 包括工作结果 (success, partial, failed) 和工作内容, 然后将工作报告返回并唤醒plan agent
plan agent开始思考worker的报告, 如果success且review合理, 就通过, 否则Plan Agent 从全局视角判断问题属于哪一类，再决定是补细节、继续分解，还是换路线
当所有worker agent完成任务且通过plan agent review的时候，plan agent生成最终的报告
